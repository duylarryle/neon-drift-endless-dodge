# Neon Drift: Endless Dodge

## Repository
https://github.com/duylarryle/neon-drift-endless-dodge/

## Description
Neon Drift is a one-screen arcade survival game where the player controls a character that must dodge an endless stream of incoming obstacles. The game gradually increases in difficulty over time, creating a fast-paced and engaging interactive experience that explores motion, timing, and visual feedback in digital media.

## Features
- Player Movement  
  - The player can move freely around the screen using keyboard controls (WASD or arrow keys) with smooth motion.

- Obstacle Spawning System  
  - Obstacles spawn at random positions and move toward the player or across the screen. The spawn system will use randomness to ensure unpredictability.

- Collision Detection  
  - The game will detect collisions between the player and obstacles using bounding boxes. A collision triggers a game-over state.

- Difficulty Scaling  
  - The game becomes progressively harder by increasing obstacle speed and spawn rate over time.

- Score System  
  - The player earns points based on survival time. The score will be displayed on screen and updated in real time.

- Game Over and Restart System  
  - When the player loses, a game-over screen is shown. The player can restart the game without closing the program.

- Visual Effects and Styling  
  - The game will feature a neon-themed visual style, including animated background elements (such as stars or glowing grids) and simple particle effects on collision.

- Optional Enhancements  
  - Additional features may include a high score system, sound effects, and different obstacle variations to improve gameplay variety.

## Challenges
- Balancing difficulty so that the game becomes more challenging without feeling unfair or overwhelming.
- Implementing smooth collision detection and object spawning without performance issues.
- Designing visually appealing effects while keeping the system efficient.
- Managing game states such as playing, game over, and restart cleanly in code.

## Outcomes
Ideal Outcome:
- A polished and visually appealing arcade game with smooth controls, progressively increasing difficulty, multiple obstacle behaviors, score tracking, and strong visual feedback.

Minimal Viable Outcome:
- A functional dodge game with player movement, random obstacle spawning, collision detection, a working score system, and a basic game over and restart feature.
